# Product Market Fit Analysis for a Brazilian Marketplace

## О проекте
Аналитический проект по оценке product-market fit маркетплейса, недавно вышедшего на рынок Бразилии.  
Бизнес-контекст: выручка маркетплейса не растёт несколько месяцев, несмотря на стабильный приток новых пользователей.

## Цель исследования
На основе имеющихся данных:

- оценить retention через когортный анализ;
- проверить наличие product-market fit;
- определить ключевые продуктовые метрики для роста прибыли;
- приоритизировать гипотезы с помощью ICE;
- предложить метрики для оценки выбранной гипотезы.

## Данные
В анализе использованы три основные таблицы:

- **customers** — идентификаторы пользователей и география доставки;
- **orders** — информация о заказах, статусах и сроках доставки;
- **order_items** — товарные позиции, цены и стоимость доставки.

## Что было сделано
В рамках проекта я:

- рассчитала месячный retention по когортам;
- оценила поведение пользователей после первой покупки;
- проанализировала динамику повторных заказов;
- проверила, могут ли логистика и отмены объяснять стагнацию выручки;
- выделила ключевые метрики для роста прибыли;
- сравнила продуктовые гипотезы с помощью ICE.

## Ключевые выводы
- У маркетплейса есть приток новых пользователей, но почти отсутствует удержание.
- Медианный retention 1-го месяца составил **0.4%**, а в следующие месяцы практически обнуляется.
- Доля клиентов с повторной покупкой — около **3%**.
- Логистика не выглядит ключевой причиной проблемы: доставка в среднем происходит вовремя или раньше обещанного срока, доля отмен и недоставленных заказов низкая.
- Рост GMV поддерживается в основном за счёт привлечения новых пользователей, а не за счёт роста LTV.
- В **90% заказов** содержится только одна товарная единица, что указывает на потенциал роста среднего чека.

## Вывод по product-market fit
На текущем этапе выраженного **product-market fit** у маркетплейса нет: продукт умеет привлекать пользователей, но не формирует устойчивую повторную ценность.

## Рекомендации
- Сфокусироваться на росте **retention** как на ключевой точке роста.
- Исследовать причины отсутствия повторных покупок: качественные интервью, сегментация ушедших клиентов, анализ сценариев первой покупки.
- Тестировать механики роста **AOV**: рекомендации, кросс-селл, порог бесплатной доставки.
- Проверить гипотезу об исправлении бага в processing заказов и отследить её влияние на долю успешно доставленных заказов.

## Ключевые метрики
Для роста прибыли продукту стоит сфокусироваться на следующих метриках:

- retention rate;
- repeat purchase rate;
- LTV;
- average order value (AOV);
- delivered orders rate.

## Приоритизация гипотез
По фреймворку **ICE** наивысший приоритет получила гипотеза об исправлении бага в processing заказов.
